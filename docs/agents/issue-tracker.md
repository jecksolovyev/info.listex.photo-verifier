# Issue tracker: None configured

This repo does **not** have an issue tracker configured. It is not a git repository and has no GitHub/GitLab remote.

## What skills should do

Skills that read from or write to an issue tracker — `to-issues`, `triage`, `to-prd`, `qa` — must **not** assume a workflow. Before acting, ask the user how they want issues handled (e.g. local markdown under `.scratch/`, or a remote tracker once one exists).

## Switching later

To adopt a tracker, re-run `setup-matt-pocock-skills`, or replace this file with one of the seed templates in
`.agents/skills/setup-matt-pocock-skills/` (`issue-tracker-local.md`, `issue-tracker-github.md`, `issue-tracker-gitlab.md`).
